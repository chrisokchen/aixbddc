# 這個目錄放什麼？

一個 boundary preset 的完整檔案家族範本：profile.yml（身分與規格綁定）、step-classification.yml（step→handler 路由）、plugin-contract.md（plan-time 履約）、shared-dsl-template.yml（kickoff 種子）、scripts/（part-to-DSL plugin），以及選用的 handlers/、variants/、dsl-arrangement-rules/、sbe-form-lock/。新增 boundary 時把整包複製成 boundaries/<your-boundary>/ 再逐檔把 `<尖括號 placeholder>` 換成真值。每個子目錄各有 .README.md 說明自己的用途與下游消費者。

# 哪裡會用到此目錄下的檔案？

boundary.yml 的 type 指向某 preset 後：aibdd-kickoff 讀 shared-dsl-template.yml、aibdd-plan 讀 profile / step-classification / plugin-contract 並 exec scripts/part_to_dsl.py、aibdd-red-execute 讀 step-classification / handlers / variants、aibdd-spec-by-example-analyze 讀 sbe-form-lock/ 與 dsl-arrangement-rules/。本 `<template>` 目錄本身永遠不會被當成 active boundary 載入。
