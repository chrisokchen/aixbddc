# `/aibdd-reconcile` Role and Contract

## Role

`/aibdd-reconcile` is the backward-entry planner for one existing AIBDD plan package.
It does not time-travel the whole repository. Instead, it classifies the earliest affected
reconcilable planner, archives the current package snapshot, cascades that planner suffix,
and records one explicit reconciliation narrative for the session.

## Reconcilable Planner Set

The only planners this skill may classify or cascade are:

1. `aibdd-discovery`
2. `aibdd-plan`
3. `aibdd-spec-by-example-analyze`

`/aibdd-tasks` and `/aibdd-implement` are never part of the cascade chain.
They become stale follow-up work after reconcile completes.

## Required Inputs

- `.aibdd/arguments.yml`
- One target `plan package` path
- One non-empty trigger description

The earliest planner is chosen by classifying the trigger description against the
ownership of the three reconcilable planners. That ownership is stated inline in the
classify phase (`02-classify-and-open-session/SOP.md`), so no external relevance
document is required:

1. `aibdd-discovery` owns which features exist, feature-file scope, and atomic-rule extraction.
2. `aibdd-plan` owns architecture/boundary, API/entity/contract design, implementation plan, and DSL synthesis.
3. `aibdd-spec-by-example-analyze` owns Gherkin example shape, step-handler mapping, DSL arrangement, and concrete parameter values.

## Outputs

- Archived snapshot under `{plan_package}/archive/<session_id>/`
- Root-level `{plan_package}/RECONCILE_RECORD.md`
- New planner outputs regenerated by the cascaded suffix
- A short user-facing report including the earliest planner, archive pointer, and follow-up reminder

## Archive Semantic

When a new reconcile session starts, the skill must move **every entry directly under the plan
package root except `archive/`** into `archive/<session_id>/`.

That means:

- archive never nests inside archive
- no cherry-pick exclusions
- the new reconcile round always starts from an empty plan-package root

## In-flight Retrigger Semantic

If the plan package already has an active reconcile session:

- do not create a new archive snapshot
- append the new trigger into the same session
- reclassify earliest planner from the merged trigger text
- compare the new earliest against the session's current cascade pointer
- if the new earliest is more upstream, replay from the new earliest
- if the new earliest is the same layer or more downstream, continue from the current pointer

## Non-Goals

- It does not rewind the whole workspace to an older checkpoint; `/aibdd-rewind` owns that job.
- It does not classify or regenerate `/aibdd-tasks` or `/aibdd-implement`.
- It does not mutate non-AIBDD product code on its own.
- It does not bypass planner ownership boundaries.
- It does not delete `archive/` history or prune old sessions.

## Completion Contract

The skill is complete when:

1. the earliest reconcilable planner has been classified;
2. a new session has either archived the package root or merged into an in-flight session;
3. the cascade suffix has been delegated through to completion;
4. `RECONCILE_RECORD.md` reflects the final session state;
5. the user has been told that `/aibdd-tasks` and `/aibdd-implement` are stale.
