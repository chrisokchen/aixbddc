## §1 API & Mock Plan

> Truth：OpenAPI。一條 `operationId` 一列。**禁止** MSW；自寫 mock 函式 + fixture 是本 package 唯一 mock 路徑。

### §1.1 Typed API Client Functions

| operationId | Method × Path | Module | Request Schema | Response Schema | Notes |
|---|---|---|---|---|---|
| `createBorrowRequest` | `POST /borrow-requests` | `src/apis/borrow-request-api.ts` | `BorrowRequestPayloadSchema` | `BorrowRequestSchema` | header 帶 `Idempotency-Key` |
| `getBorrowRequest` | `GET /borrow-requests/{id}` | 同上 | – | `BorrowRequestSchema` | 404 走 `not-found.tsx` |
| `listBorrowRequests` | `GET /borrow-requests` | 同上 | `ListQuerySchema` | `Page<BorrowRequestSchema>` | server-side cursor |

### §1.2 Zod Schemas

| Schema | File | OpenAPI `$ref` |
|---|---|---|
| `BorrowRequestSchema` | `src/lib/schemas/borrow-request.ts` | `#/components/schemas/BorrowRequest` |
| `BorrowRequestPayloadSchema` | 同上 | `#/components/schemas/BorrowRequestPayload` |
| `ListQuerySchema` | `src/lib/schemas/list-query.ts` | shared |

> `z.infer<typeof XSchema>` 為 §2/§3 的 props / state 型別 SSOT；**禁止**手寫對應 `interface`。

### §1.3 Mock Implementations（custom — NO MSW）

> 每個 mock 是純 typed function，回傳 fixture-shaped data；同時供 §2 Storybook stories 與 §3 e2e fixtures 消費。**禁止**任何 `setupWorker` / `mockServiceWorker.js` 引入。

| Mock fn | File | Backs operationId | Fault scenarios |
|---|---|---|---|
| `mockCreateBorrowRequest` | `src/mocks/borrow-request.ts` | `createBorrowRequest` | 200 success / 400 validation / 409 duplicate-id |
| `mockGetBorrowRequest` | 同上 | `getBorrowRequest` | 200 / 404 |
| `mockListBorrowRequests` | 同上 | `listBorrowRequests` | empty / 1 page / multi-page |

| Fixture builder | File |
|---|---|
| `borrowRequestFixture(overrides?)` | `src/mocks/fixtures/borrow-request.ts` |

### §1.4 Wire-up

- Test / dev mode：API client 透過 `src/lib/api-client.ts::resolveTransport()` 讀 `NEXT_PUBLIC_API_TRANSPORT=mock`，在 process 內把 `operationId → mock fn` 直接路由。
- Production build：以 build-time guard（`process.env.NODE_ENV === 'production'` 檢查）拒絕 `src/mocks/**` 被 import；漏網 → bundle analyzer 在 CI 攔截（規則化在 §4 補）。

## §3 Page Plan

> Truth：`${FEATURE_SPECS_DIR}/*.feature`。一條 route 一列；scenario reference 為合法紅燈的對映依據。

### §3.1 Route Map

| URL | File | Server / Client | Composes (§2) | Calls (§1) | Backing scenarios |
|---|---|---|---|---|---|
| `/borrow-requests/new` | `src/app/borrow-requests/new/page.tsx` | client（表單） | `BorrowRequestForm` | `createBorrowRequest` | `01-提交借閱申請.feature::S1/S2` |
| `/borrow-requests` | `src/app/borrow-requests/page.tsx` | server + client subtree | `BorrowRequestList`, `StatusBadge` | `listBorrowRequests` | `02-列出借閱申請.feature::S1` |
| `/borrow-requests/[id]` | `src/app/borrow-requests/[id]/page.tsx` | server | `StatusBadge`（+ 詳情區） | `getBorrowRequest` | `03-檢視借閱申請.feature::S1/S2` |

### §3.2 Server / Client boundary policy

- 預設 **server component**；只有需要互動 state 的 subtree 才 `'use client'`。
- API client **僅**在 server component 直接呼叫；client component 透過 props / route segment fetch 取得資料。
- `env.ts`（`src/lib/env.ts`）只在 module top-level 消費；**禁止**在 render body 內讀 `process.env`。

### §3.3 Route-level UX hooks

| Concern | Handling |
|---|---|
| Loading | `loading.tsx` per segment |
| Error | `error.tsx` per segment，使用 `getErrorMessage(error)` 安全 narrow |
| Not found | `not-found.tsx` for `/borrow-requests/[id]` |
| Redirect after submit | client component → `useRouter().push('/borrow-requests')` |
