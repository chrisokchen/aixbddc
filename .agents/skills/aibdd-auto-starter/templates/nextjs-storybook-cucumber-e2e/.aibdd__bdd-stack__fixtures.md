# Fixtures Runtime

> v2 (2026-05-10): mock 控制面從「`mockControl` HTTP client → `/__test__/*` Route Handler」改為「`mockApi` closure-local fixture + `page.route` 攔截」。fixture 內**不再有任何 HTTP 跨 process call**；mock state 住在 fixture closure，DevTools protocol 由 Playwright 在 browser ↔ test-runner 之間架橋。

## fixture shape

playwright-bdd fixtures 由 `features/steps/fixtures.ts` 透過 `test.extend(...)` 宣告。本 stack 的 fixture surface（v2）：

```ts
/* eslint-disable react-hooks/rules-of-hooks -- Playwright fixture `use` callback is not a React Hook */
import { test as base, createBdd } from 'playwright-bdd';

interface MockApi {
  reset(): void;
  // domain-specific seeders, 例：
  seedRoom(input: { roomCode: string; seatsTaken: number; capacity?: number }): void;
  inspectRoom(code: string): RoomState | undefined;

  // Tier-2 only:
  // override(operationId: string, response: OverrideResponse, sequence?: number): void;
  // calls(operationId: string, since?: string): RecordedCall[];
}

interface OverrideResponse {
  status: number;
  body: unknown;
  headers?: Record<string, string>;
}

interface RecordedCall {
  operationId: string;
  request: { method: string; url: string; headers: Record<string, string>; body: unknown };
  response: { status: number; body: unknown };
  schemaValid: boolean;
}

const API_HOST = 'http://localhost:4000'; // 必與 Next.js dev host 不同

export const test = base.extend<{ mockApi: MockApi }>({
  mockApi: async ({ page }, use) => {
    const store = new Map<string, RoomState>();      // closure-local mock state
    // const overrides = new Map<string, ...>();      // Tier-2: 行為 override
    // const calls: RecordedCall[] = [];              // Tier-2: call recorder

    await page.route(`${API_HOST}/<resource>/**`, async (route) => {
      const req = route.request();
      const url = new URL(req.url());
      const method = req.method();
      // 1. Match method × url.pathname.
      // 2. （optional Tier-2）讀 overrides 決定下一次回應；否則走預設 happy-path。
      // 3. Mutate `store` for write operations.
      // 4. Build response body conforming to its responseSchema (Zod parse before fulfill).
      // 5. （optional Tier-2）push 到 calls。
      // 6. route.fulfill({ status, contentType: 'application/json', body: JSON.stringify(...) }).
    });

    const api: MockApi = {
      reset: () => store.clear(),
      seedRoom: (input) => store.set(input.roomCode, /* build entity */),
      inspectRoom: (code) => store.get(code),
    };
    await use(api);
  },
});

export const { Given, When, Then } = createBdd(test);
```

`mockApi` 為**同步**介面：seed / inspect / reset 皆直接讀寫 closure 內 `store`，不走 HTTP、不 await。step defs 呼叫時不需 `await`。

walking skeleton 起始 fixtures.ts 是空殼（`Fixtures = Record<string, never>`），含 commented-out skeleton（v2 starter template 已預載）；第一個 feature package 落地時 **必須** 升級為含 `mockApi` 的完整定義。升級不在 starter 自動完成 — 由 `/aibdd-red-execute` Phase 6 在第一條 `mock-state-given` / `api-stub` step 出現時觸發。

可選 fixture（按需擴）：

- `clock` — Playwright `page.clock` 包裝；只在某 package 啟用 `time-control` handler 時加入。
- `mcpProbe` — Storybook MCP query helper；只在 design-system component 進入 `L4.source_refs.component` 時加入。
- `viewportProfile` — 命名 viewport 切換 helper；只在某 package 用 viewport 名稱（mobile / tablet / desktop）作參數時加入。

## data source

- 跨 process 邊界唯一通道：Playwright `page.route`（DevTools protocol）。fixture 註冊 route，Playwright 把 browser 端 fetch 轉到 test-runner process（boundary invariant I1）。
- mock store 真值：fixture closure 內的 `Map<>` / object / class；**不**穿過 process 邊界、**不**對外 import。
- mock 行為 override（Tier-2）：closure 內的 `Map<operationId, response[]>`，由 `mockApi.override(...)` 同步入隊。
- call recorder（Tier-2）：closure 內的 array，每次 `route.fulfill` 前 push。
- response shape 來源：對應產品端 schema 的 Zod schema（`src/lib/schemas/**/*.ts`），由 fixture handler 在 `route.fulfill(...)` 前以 `responseSchema.parse(...)` 自動 enforce（I2）。
- 視覺合約：Storybook story export 的 `args` —— 該合約**不**穿過 fixture，由 `step-definitions.md` 的 locator derivation rules 直接派生。

## reset policy

- 每個 scenario 的**自動 reset 入口**：fixture scope = test → closure 在每個 scenario 開始時被 Playwright 重建（I3）。
- 不需要呼叫 `mockApi.reset()`；它存在只是給少數 scenario 內想中途清空時用（罕見）。
- **禁止** 在 step file 內補額外 `Before` / `After` hook 變更 closure state；fixture 自動重建是唯一合法路徑。
- Playwright project（chromium / firefox / webkit）切換不依賴 webServer 重啟做隔離；fixture closure 為 per-test 隔離，**並行 workers 安全**（無共享狀態）。
- 識別碼漂移（auto-increment / UUID）若 acceptance 斷言依賴 deterministic id → 由 `mockApi.seedXxx(...)` 顯式提供 fixture，**不**靠 reset 做。

## API host separation

- `NEXT_PUBLIC_API_BASE_URL`（讀取自 `.env.local`，預設 `http://localhost:4000`）的 host **必須**與 Next.js dev server host（`http://localhost:3000`）不同。
- fixture 之 `page.route(API_HOST + '/<path>', ...)` 以該 API host 為前綴；page navigation 走 dev host 時不會被攔截。
- `playwright.config.ts.webServer.env` **不得**覆寫 `NEXT_PUBLIC_API_BASE_URL`；如被覆寫成 dev host，`page.route` 會把 page navigation 一起 intercept，handler 強行回 JSON 會破壞頁面渲染。
- 不對 → `prehandling-before-red-phase.md` §3.7 fail-stop。

## missing-truth routing

任何必要 fixture truth 缺漏 → **不**在 fixture 內以寬鬆 mock 補位；STOP，並 route 對應上游 skill：

| 缺漏項 | route 到 |
|---|---|
| `mockApi` 介面方法缺 / fixtures.ts 仍是空殼 | `/aibdd-red-execute` Phase 6（第一條 mock-state-given step 觸發升級） |
| `page.route` handler 對應 operation 缺 | `prehandling-before-red-phase.md` §3.3 → §4.2 補 fixture handler |
| Story export 缺 a11y args | Story 作者（manual gate） |
| OpenAPI 缺 operation | `/aibdd-form-api-spec`（contract-first profile only） |
| Zod schema 缺 | `prehandling-before-red-phase.md` §4.1 |
| API host 衝突 | `prehandling-before-red-phase.md` §3.7 → §4.7 |

缺 fixture truth 不是合法紅燈；屬基礎設施紅，由 `prehandling-before-red-phase.md` 阻擋於 RGB cycle 之外。

## known limitations

- **檔案上傳 fixture** 未 scaffold；`page.setInputFiles(...)` + 對應 `page.route` handler 需要時自行擴。
- **瀏覽器儲存 fixture**（cookies / localStorage / sessionStorage）為 Tier-3 future expansion；目前若 scenario 需要 auth pre-seed，請走 Playwright `storageState` 一次性 login 機制，不要寫進 BDD step。
- **網路節流 / offline fixture** 為 Tier-3；Playwright `page.context().setOffline(true)` 不在 starter 範圍。
- **多 tab / popup** 場景 starter 無 fixture；首次需要時擴 `tabContext` fixture，並更新 `mockApi` 以區分 per-tab calls（注意 `page.route` 是 per-page，多 tab 時要在每個 page instance 都註冊）。
- **多 baseURL**（A/B、microfrontend）目前不支援；單一 Next.js dev baseURL + 單一 API host 為 v1 假設。
- **chromium-only 加速** 模式預設關閉；若想本地只跑 chromium，用 `--project=chromium`，**不**改 `playwright.config.ts.projects` 永久刪除其他瀏覽器（會破壞 CI 假設）。
