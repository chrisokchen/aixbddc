# Acceptance Runner Runtime

## acceptance runner command

```bash
npm run test:e2e
```

等價於 `bddgen && playwright test`：

1. `bddgen`（playwright-bdd CLI）讀 `playwright.config.ts` 中 `defineBddConfig(...)` 設定，把 `*.feature` 編譯成 Playwright spec 檔（預設輸出在 `.features-gen/`）。
2. `playwright test` 啟動 `webServer`（`npm run dev`），對 `baseURL` 跑 generated spec。

跑單一 feature 或 scenario：

```bash
npx bddgen && npx playwright test path/to/scenario.spec.ts
# 或 by tag
npx bddgen && npx playwright test --grep "@smoke"
```

跑單一 project（瀏覽器）：

```bash
npx bddgen && npx playwright test --project=chromium
```

## dry-run / discovery command

```bash
npm run bddgen
npx playwright test --list
```

`bddgen` 自身就是 BDD 層 dry-run 等價：失敗即 step pattern 對不上 / fixture 缺漏 / config 不一致；輸出 `.features-gen/` 即視為 BDD layer 通過。
`playwright test --list` 接著確認 spec 對齊 + 不執行 webServer，等同 Playwright runner discovery。

## runner config file path

`playwright.config.ts`

關鍵欄位：

- `defineBddConfig({ features, steps })` — feature 與 step glob 的 SSOT；實際數值由 `step-definitions.md` 規定。
- `use.baseURL` — 預設 `process.env.BASE_URL ?? 'http://localhost:3000'`。
- `webServer.command` — `npm run dev`。**不**設 `webServer.env`（v2: 不再有 transport switch；mock 由 fixture `page.route` 攔截，dev server 跑普通模式即可。覆寫 `NEXT_PUBLIC_API_BASE_URL` 為 dev host 會違反 §3 API host separation）。
- `projects` — `chromium` / `firefox` / `webkit`。
- `reporter` — `list` + `cucumberReporter('html', { outputFile: 'cucumber-report/index.html' })`。

## runtime feature glob

由 `playwright.config.ts::defineBddConfig.features` 宣告；當前 starter 預設值為 `features/*.feature`。實際 layout 規則（含 `features/<package>/<sub>/*.feature` 等）以 `feature-archive.md` 為準。

## step definition glob

由 `playwright.config.ts::defineBddConfig.steps` 宣告；當前 starter 預設值為 `features/steps/*.ts`。實際 layout 規則（含 `features/steps/<package>/<handler>/*.ts`）以 `step-definitions.md` 為準。`acceptance-runner.md` 不複寫該 SSOT。

## shared step glob（若有）

無獨立 glob；shared matcher 仍落在 `defineBddConfig.steps` 涵蓋範圍內。命名與唯一性由 `step-definitions.md` 規約，acceptance-runner 不另外開新 glob。

## report output path

- HTML cucumber report：`cucumber-report/index.html`（由 `cucumberReporter` 產出）。
- 失敗附件：`screenshot: 'only-on-failure'`、`video: 'retain-on-failure'`、`trace: 'on-first-retry'`，由 Playwright 預設輸出到 `test-results/`。
- 開啟最近一次報告：`npm run test:e2e:report`（= `playwright show-report`）。

## runner visibility check 方法

1. 確認 `playwright.config.ts` 的 `defineBddConfig.features` / `defineBddConfig.steps` 涵蓋所有目標 feature 與 step file。
2. 執行 `npm run bddgen`；無錯誤輸出且 `.features-gen/` 內含對應 spec 即 BDD 層可見。
3. 執行 `npx playwright test --list` 列出所有發現的 test，逐條與目標 Scenario 對表。
4. 若 dry-run 缺 feature 或 step → 修 runner wiring（feature glob / step glob / `bddgen` config）；**不**進入紅燈計次。
5. 若 dry-run 通過但 `npm run test:e2e` 在 webServer 啟動階段失敗（port 占用、`.env` 缺鍵、`npm run dev` 失敗）→ 走 `prehandling-before-red-phase.md` §4.0 環境前置流程，**不**視為 RGB 失敗。

## stack-specific caveat / known limitations

- `bddgen` 不執行 `features/steps/fixtures.ts` 的 `test.extend(...)` runtime 副作用；它只做 step-pattern 匹配與 spec 產出。fixture 真正生效在 `playwright test`。
- `webServer.reuseExistingServer: !process.env.CI`：本地會復用既有 dev server。v2 mock 由 fixture `page.route` 攔截，dev server 是否「正確啟動」與 mock 無關 —— 只要 dev server 起得來就行；若 dev server 起不來，走 `prehandling-before-red-phase.md` §4.0 流程處理。
- Playwright 1.45+ `page.clock` 是 `time-control` handler 的唯一合法時鐘控制；**禁止** `Date.now()` mock 或 `setTimeout` 包裝。
- chromium / firefox / webkit 三 project 同跑時，每個 project 都會獨立啟一輪 webServer；scenario 之間的 mock state 隔離由 fixture closure 自動處理（fixture scope = test，boundary invariant I3），**不**依賴 webServer 重啟、**不**需手動 reset。
- `cucumberReporter('html')` 把每條 scenario 轉成 cucumber-format HTML；attachments（screenshot / video / trace）以 `externalAttachments: true` 引用 `test-results/` 內檔案，不嵌入 HTML 主文件。
- 中文 feature 檔名 / Scenario 名依賴本機檔案系統 + Node encoding；macOS / Linux 預設 UTF-8 沒問題，Windows 須確認 `chcp 65001`。
