# Step Definitions Runtime

## primary step definition root

`features/steps`

## step definition glob

由 `playwright.config.ts::defineBddConfig.steps` 宣告。

- starter 預設值：`features/steps/*.ts`（flat，僅供 walking skeleton 的 8 條 baseline shared step 使用）。
- 推薦值（第一個 feature package 落地前必須擴）：`features/steps/**/*.ts`，以涵蓋 nested handler layout。
- `acceptance-runner.md` 不複寫此 glob；本檔為 SSOT。

## shared step glob（若有）

無獨立 glob；shared matcher 仍落在 `defineBddConfig.steps` 涵蓋範圍內。

shared step 落點：

- baseline shared（starter ships，跨 package 通用 8 條）：`features/steps/home.steps.ts`
- package-internal shared（同一 package 跨 scenario 共用，但不跨 package）：`features/steps/<function-package-or-domain>/_shared/*.ts`
- cross-package shared（罕見，需審）：`features/steps/_shared/*.ts`

唯一性 guard：同一 matcher pattern 字串只能 register 一次；`bddgen` 在 step-pattern 衝突時會 fail-stop，**不**自動裁定優先順序。

## handler layout

Generated feature-package steps **必須** 落在下列目錄：

```text
features/steps/<function-package-or-domain>/
  route-given/                  # Tier-1
  viewport-control/             # Tier-1
  mock-state-given/             # Tier-1
  time-control/                 # Tier-1
  ui-action/                    # Tier-1
  operation-response-success-and-failure/  # Tier-1
  ui-readmodel-then/            # Tier-1
  api-stub/                     # Tier-2 — 僅當 test-strategy.yml 啟用
  url-then/                     # Tier-2
  api-call-then/                # Tier-2
  mock-state-then/              # Tier-2
```

對映 SSOT：本 repo 的 [`nextjs-playwright.md`](../../../../aibdd-core/assets/boundaries/web-frontend/variants/nextjs-playwright.md) Step File Layout（11 個 handler id）。

規則：

- 目錄名 = handler id（連字號保留，不改成 underscore）。
- 一個 generated step pattern 對應一個 `.ts` 檔；shared matcher 例外見上。
- Tier-2 handler 目錄**只在** `${TEST_STRATEGY_FILE}.tier2_handlers.<handler> == enabled` 時建立；未 enable 卻出現對應目錄 → `prehandling-before-red-phase.md` §3.6 阻擋。

## matcher rules

- **必須** 從 `features/steps/fixtures.ts` 匯入 `Given` / `When` / `Then`：
  ```ts
  import { Given, When, Then } from './fixtures';   // 或相對路徑回到 fixtures.ts
  ```
- **禁止** 直接 `import { createBdd } from 'playwright-bdd'`；二次呼叫 `createBdd(test)` 會回傳新實例，破壞 fixture 共享，出現 `mockApi` 為 `undefined` 等難 trace 錯誤。
- 第一個 fixture 參數固定 destructure：`{ page, request, mockApi, ... }`；額外位置參數對應 L1 placeholder。
- placeholder 對映：
  - integer → `{int}`
  - float → `{float}`
  - string（matcher 內以雙引號包住）→ `{string}`
- matcher 字面值 = DSL 對應 entry 的 `L1`，**逐字一致**（含標點、空白、引號）。`/aibdd-red-execute` 由 L1 推 matcher 字串，**不**手寫不等價變體。

## locator derivation rules（for `ui-action` / `ui-readmodel-then`）

- locator 派生來源：`L4.source_refs.component` 指向的 Story export 之 `args`（boundary invariant I4）。
- 派生順序：`getByRole(role, { name })` → `getByLabel(name)` → `getByTestId(testId)`；前項可解則優先用，後項僅在前項無法表達時回退。
- **禁止** raw CSS class selector（如 `.btn-primary`）、nth-child positional selector（如 `nth=2`）、xpath 字面值。
- design-system component **必須** 先呼 `${PROJECT_SLUG}-sb-mcp` 的 `list-all-documentation` → `get-documentation` 確認 prop 與 accessible name；MCP 不可用 → fail-stop（見 `prehandling-before-red-phase.md` §5）。

## cross-process surface rules（for `mock-state-*` / `api-*`）

- 所有 `mock-state-given` / `mock-state-then` / `api-stub` / `api-call-then` step **必須** 透過 fixture 暴露的 `mockApi` 介面呼叫；介面 SSOT 在 `fixtures.md`。
- **禁止** 從 step file 直接 `import` `src/mocks/**`（v2 不應存在）或產品端 module 作為 mock store 來源。
- **禁止** 從 step file 內呼叫 `page.route(...)` 攔截 fixture `mockApi` 已涵蓋的 operation —— 會分裂 truth；fixture 是唯一 page.route 註冊點。
- mock layer 跨 process 通道由 fixture 內 `page.route` 宣告（DevTools protocol）；step file **不**自註 page.route、**不**走 HTTP 呼叫 mock，全部走 `mockApi` 同步介面。

## starter baseline step inventory（home.steps.ts，跨 package shared）

walking skeleton 預先提供 8 條 shared step，對映 web-frontend handler 如下：

| matcher | handler | 來源 |
|---|---|---|
| `Given I am on the home page` | route-given | `home.steps.ts` |
| `When I resize the viewport to {int} x {int}` | viewport-control | `home.steps.ts` |
| `Then the page title should be {string}` | ui-readmodel-then | `home.steps.ts` |
| `Then I should see the heading {string}` | ui-readmodel-then | `home.steps.ts` |
| `Then I should see the link {string}` | ui-readmodel-then | `home.steps.ts` |
| `Then I should see the button {string}` | ui-readmodel-then | `home.steps.ts` |
| `Then I should see the text {string}` | ui-readmodel-then | `home.steps.ts` |
| `Then the link {string} should point to a URL containing {string}` | ui-readmodel-then | `home.steps.ts` |

這 8 條 **不得** 在 package-internal step 重新 register（會踩 step-pattern 衝突）；要客製化請改 L1 文字並另外建 matcher。

## forbidden

- 直接 `import { createBdd } from 'playwright-bdd'` —— 必須走 `./fixtures`。
- 從 `src/app/**` 或 component 內部模組 `import` —— 觀察 surface 限定 DOM / `mockApi`。
- 從 `src/mocks/**` import 任何模組 —— v2 此目錄不應存在；mock state 在 fixture closure。
- 在 step file 內 `await new Promise(r => setTimeout(r, ...))` 真實等待時間 —— 時間控制走 `page.clock`（見 `time-control` handler）。
- 在 step file 內 register 額外 `Before` / `After` hook 變更 fixture closure state —— fixture scope 自動 reset 是唯一合法路徑（boundary invariant I3）。
- 在 step file 內呼叫 `page.route(...)` 攔截 fixture `mockApi` 已涵蓋的 operation —— 會分裂 truth；fixture 是唯一 page.route 註冊點。
- raw locator string（CSS class、nth-child、xpath）—— 違反 I4 locator derivation。
- 在同一 package 重新 register 上面 starter baseline 的 8 條 shared matcher —— step-pattern 衝突。
- 跳過 BDD 直接寫 `test('...', ...)` raw Playwright test file —— 違反 `dev-constitution.md` §6 raw test opt-out 條款。
