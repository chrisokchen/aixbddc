# Dev Constitution — Next.js + Storybook + playwright-bdd Frontend (Starter)

> v2 (2026-05-10): Mock layer 從「dev-server in-process mock + `/__test__/*` HTTP 端點 + `NEXT_PUBLIC_API_TRANSPORT` switch」改為「Playwright fixture closure + `page.route` 攔截」。`src/mocks/**` 與 `src/app/__test__/**` 為 deprecated，從 walking skeleton 與所有產品端 module 移除。

## §1 Layer Ownership（精簡）

| 層 | 慣例路徑 | 擁有 | 禁止 |
|---|---|---|---|
| App (routes) | `src/app/**` | App Router page / layout / loading / error / not-found，server vs client 邊界決定 | 直接 import 元件內部模組、繞過 `lib/api/` 自造 fetch |
| Components | `src/components/**/<X>/<X>.tsx` | feature 元件、UI primitive、與 Story 對偶的 props shape；Story colocate 同層 | 跨層讀 `src/app/**`、直接呼 `lib/api/*`（須由 page 或 hook 注入） |
| Hooks | `src/hooks/**` | custom React hooks（`useRoom` / `useGuess` 等），polling、derived state、side-effect orchestration | 直接 render JSX、長住業務規則（規則屬 `lib/domain` 或 server） |
| APIs | `src/lib/api/**/<aggregate>.ts` | typed API client function（一個 aggregate 一檔），呼叫 `lib/api-client` 派送；pure `apiFetch(...)` + Zod parse | 自寫 fetch、retry / cache / auth-renewal / 業務分支（屬 Green 範圍）、繞過 `lib/schemas` 自定 type |
| Lib | `src/lib/**` | `api-client.ts`（純 fetch wrapper，**無 transport switch**）、`env.ts`（typed env）、`schemas/`（Zod SSOT）、`api/`（per-aggregate client）、純工具 | 任何 React JSX、任何 feature-specific 規則、`mockOperationId` 之類 mock-aware 參數 |
| Stories | `src/components/**/<X>/<X>.stories.@(ts\|tsx)` | Storybook story export（**元件合約 SSOT**），`args` 必含 accessible-name；colocate 於元件同層 | 在 story 內手寫 inline literal data（必須走 fixture / static factory）、集中式 `src/stories/`（v2 deprecated） |
| Features | `features/**` | playwright-bdd feature file × step definition × fixtures（**mock layer SSOT**） | 從 step file 直接 import `src/app/**` 或 component 內部 |
| Storybook config | `.storybook/**` | Storybook 9+ entry / preview / addon-a11y / addon-vitest 設定 | 功能專屬 args 寫死 |

**已移除（v2 deprecated）**：

- ~~`Mocks: src/mocks/**`~~ — mock 不在產品端；改住 `features/steps/fixtures.ts` closure。
- ~~`Test hook route: src/app/__test__/**`~~ — 不再需要 dev-server 端 HTTP mock control。

### §1.1 App Entry Contract

- `src/app/layout.tsx` **必須** 為 root layout，掛載 `<html lang>` 與全域 CSS。
- `src/app/page.tsx` **必須** 存在以滿足 Next.js `/` route 健康狀態。
- `src/app/__test__/**` **禁止存在**（v2 deprecated）。

## §2 Dependency Direction

**允許**

```text
app -> components
app -> hooks                           # container page 用 hook 抓資料 / 訂閱狀態
app -> lib/api                         # server component 直接呼 per-aggregate api function
app -> lib/{api-client, env, schemas}
hooks -> lib/api                       # hook 內呼 api function
hooks -> lib/schemas                   # 型別 only
components -> lib/schemas              # 型別 only，不可呼叫 api-client / lib/api
features/steps -> features/steps/fixtures
features/steps -> mockApi (closure-local; in-process via page.route)
features/steps/fixtures -> lib/schemas # response shape validation (I2)
lib/api -> lib/api-client              # per-aggregate function 透過 wrapper 派送 fetch
lib/api -> lib/schemas                 # request / response Zod parse
stories -> components                  # 視覺合約；資料用 static factory 或 storybook decorator
```

**禁止**

```text
components -> app                      # 子層不可反向 import
components -> lib/api                  # 由 page 或 hook 注入，不在 component 內 fetch
components -> hooks/*                  # 元件保持 presentational；hook 由 container 注入結果
lib -> components / hooks / app        # lib 是底層，不依賴上層
src/mocks/** -> 任何 import 點         # v2: 不應存在
features/steps -> src/mocks/**         # 同上；mock 在 fixture 內
features/steps -> src/app              # 透過 DOM / mockApi 觀察，不直連 internals
features/steps -> components/*.tsx     # 同上；只能透過 rendered DOM
features/steps/fixtures -> src/lib/api-client  # fixture 不直接呼產品 fetch；只攔截
src/app/__test__ -> 任何路徑           # v2: 不應存在
src/stories/**                         # v2 deprecated；Story colocate 於 components/<X>/
```

## §3 Configuration And Dependency Injection

- 環境變數**必須**經 `src/lib/env.ts` typed access，不得在 render body 或 component 內讀 `process.env`。
- `src/lib/api-client.ts` 為**純 fetch wrapper**：讀 `env.NEXT_PUBLIC_API_BASE_URL` 解析 base URL，不含 transport switch、mock dispatch 路徑、`mockOperationId` 等 mock-aware 參數。產品端**永遠**走真實 fetch；mock 由 fixture 端 `page.route` 攔截。
- **禁止**在 feature 模組建立全域 API client singleton；唯一 entry 為 `src/lib/api-client.ts`。
- Server component **僅**在 module top-level 讀環境變數；client component 取值需透過 props / route segment data。
- **API host separation 強制**：`NEXT_PUBLIC_API_BASE_URL` 之 host **必須**與 Next.js dev server host 不同（例：dev = `http://localhost:3000`、API = `http://localhost:4000`），避免 `page.route` glob 同時 intercept page navigation。
- `playwright.config.ts.webServer.env` **不得**覆寫 `NEXT_PUBLIC_API_BASE_URL` 為 dev server host；建議完全不設 `webServer.env`，讓 `.env.local` 預設生效。

## §4 Mock Layer And Contract Sync

**Mock layer SSOT（v2）**：`features/steps/fixtures.ts` 內的 `mockApi` fixture。

- mock 狀態（rooms / orders / sessions / 任何 entity 集合）**必須**住在 fixture 的 closure（`Map<>` / object / class）；fixture scope = test → 每個 scenario 自動重建（boundary invariant I3）。
- 對外暴露純 in-process 控制 API：`mockApi.{seedXxx, reset, inspectXxx, override, calls}`，**同步**呼叫；step defs 直接 mutate 不繞 HTTP。
- API 攔截透過 `page.route(API_HOST + '/<path>', handler)`：handler 內依 `req.method()` × path 解析 dispatch；`route.fulfill({ status, contentType: 'application/json', body: JSON.stringify(...) })` 回應。
- 每個產品端 fetch 之 operation **必須**對應一個 `page.route` 匹配；缺者由 `prehandling-before-red-phase.md` §3.3 阻擋。
- handler 回傳 JSON **必須**能被對應 `responseSchema.parse(...)` 通過；I2 schema gate 由 fixture handler 在 `route.fulfill` 前 enforce。
- 每個 OpenAPI / 產品端 schema **必須**對應一個 Zod schema（`src/lib/schemas/**/*.ts`）；`z.infer<typeof XSchema>` 為下游唯一 typed entry，**禁止**手寫對應 `interface`。
- **禁止** `setupWorker` / `mockServiceWorker.js` / MSW 引入。
- **禁止** `src/mocks/**` 重新出現。
- **禁止** 把 mock store 暴露給產品端 import；store 是 test-process-only。
- **禁止** 在 step file 內額外 `Before` / `After` hook 變更 closure state；fixture 自動 reset 是唯一合法路徑。

## §5 （已移除 — v2 deprecated）

舊 §5 為 `/__test__/*` Route Handler 的 production guard。v2 page.route 架構不需要此 handler，guard 條款隨之失效。

`src/app/__test__/**` 在 v2 為**禁止存在**；CI build path 不需要特殊 guard，但 build 時應檢查無漏網（grep `__test__` 字面值出現 → fail build）。

## §6 Testing Layer Contract

| 層 | 工具 | 範圍 | 不做的事 |
|---|---|---|---|
| Component | Storybook + addon-a11y + addon-vitest interactions | 元件級不變數（a11y / prop validation / render correctness / 互動 happy + 失敗各 ≥ 1） | E2E 範圍的跨頁流 |
| Unit | Vitest | 純工具、reducer、derive 函式、Zod schema 驗證行為 | UI rendering（交給 Storybook） |
| E2E | playwright-bdd × Playwright | composition、cross-page flow、與 mock backend 的整合（mock 由 fixture page.route 提供） | 元件內部 focus ring / animation curve（信任 Storybook） |

- Storybook 為**元件合約 SSOT**；E2E feature scenario 不複寫元件級不變數。
- **禁止** raw test file opt-out — 所有 acceptance test 必須由 `.feature` × DSL × handler routing 推導；不允許「跳過 BDD 直接寫 Playwright `test()`」。
- E2E 預設 `fullyParallel: true`；fixture closure 為 per-test 隔離，**不需要** `--workers=1`。
- 視覺回歸（Chromatic / test-runner）為 deferred；不在 walking skeleton C-lite scope。
