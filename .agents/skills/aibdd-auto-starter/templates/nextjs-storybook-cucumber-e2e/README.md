## AIBDD Workflow

This skeleton expects AIBDD tooling:

1. `/aibdd-flows-specify` — capture requirements as activities + features
1. `/aibdd-rules-specify` — capture requirements as atomic rules in features
2. `/aibdd-form-api-spec` — derive `api.yml` from contracts
3. `/aibdd-auto-frontend-msw-api-layer` — generate Zod schemas, MSW handlers, API client functions
4. `/aibdd-auto-frontend-nextjs-pages` — drive page implementation from `.activity` + `.feature` truth

Empty `.gitkeep` placeholders mark the canonical layer locations: `src/lib/schemas/` (Zod SSOT), `src/lib/api/` (per-aggregate `apiFetch` functions), `src/components/` (colocated stories), `src/hooks/` (custom React hooks). `/aibdd-plan` × Pre-Red gate fills them as features land.

## Conventions

- `@/*` import alias maps to `src/*`
- API client functions live in `src/lib/api/<aggregate>.ts`; each calls `apiFetch(path, { method, body, schema })` from `src/lib/api-client.ts` (centralised wrapper handles base URL, headers, `!res.ok → ApiError`, Zod parse)
- Zod schemas in `src/lib/schemas/` are the typed SSOT; `z.infer<typeof X>` is the only typed entry — do not hand-write parallel `interface`
- Server / client env is split via `src/lib/env.ts`
- Storybook stories colocate with components: `src/components/<X>/<X>.stories.@(ts|tsx)` (centralised `src/stories/` is v2 deprecated)
- Cucumber `Given/When/Then` come from `features/steps/fixtures.ts` (`createBdd(test)`)
