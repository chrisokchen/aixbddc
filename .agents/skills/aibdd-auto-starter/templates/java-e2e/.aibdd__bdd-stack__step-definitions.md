# Step Definitions Runtime

## primary step definition root

`src/test/java/${BASE_PACKAGE_PATH}/steps`

## step definition glob

`src/test/java/${BASE_PACKAGE_PATH}/steps/**/*.java`

## shared step glob（若有）

`src/test/java/${BASE_PACKAGE_PATH}/steps/common_then/**/*.java`

## handler layout

Generated feature-package steps should be grouped under:

```text
src/test/java/${BASE_PACKAGE_PATH}/steps/<function-package-or-domain>/
  aggregate_given/
  http_operation/
  success_failure/
  readmodel_then/
  aggregate_then/
  time_control/
  external_stub/
```

## matcher rules

- Use Cucumber 註解：`@Given`、`@When`、`@Then`（套件 `io.cucumber.java.en.*`，與 starter 內 `HealthSteps` 一致）。
- 共用狀態經 `ScenarioContext`（`@Component @ScenarioScope`）注入，**禁止**以 static field 或 `ThreadLocal` 私存。
- HTTP 互動採 `MockMvc`（由 `CucumberSpringConfiguration` 的 `@AutoConfigureMockMvc` 注入）；**禁止**在 step 中 `new TestRestTemplate()` 或自啟 Tomcat。
- Matchers 來自 DSL `L1` 句型，與 Discovery 產出的 `.feature` 一致。

## forbidden

- Do not import production service internals in E2E step definitions（透過 Controller→Service→Repository 依賴方向，僅由 MockMvc 觸發）。
- Do not create step files outside the configured globs（要新增就放回 `steps/<package>/`）。
- Do not generate raw locator or UI-oriented steps for this backend stack。
- Do not 改寫 starter 內 `CommonThen` 的中文 step pattern（`操作成功`／`操作失敗`／`錯誤訊息應為 {string}`／`操作失敗，原因為 {string}`）；其為共用 SSOT，新增 reason 變體請於同檔擴充。
