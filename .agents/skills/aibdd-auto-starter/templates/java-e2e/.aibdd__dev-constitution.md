# Dev Constitution — Java Spring Boot Backend (Starter)


## §1 Layer Ownership（精簡）

| 層 | 慣例路徑 | 擁有 | 禁止 |
|---|---|---|---|
| Controller | `src/main/java/${BASE_PACKAGE_PATH}/controller` | REST endpoint、HTTP 狀態對應、DTO 進出轉譯 | 略過 service 直連 DB、業務分支 |
| Service | `src/main/java/${BASE_PACKAGE_PATH}/service` | 用例編排、交易意圖、領域決策 | HttpServletRequest/Response、裸 SQL |
| Repository | `src/main/java/${BASE_PACKAGE_PATH}/repository` | 查詢／寫入（基於 Spring `JdbcClient`） | HTTP 外形、非儲存必需的業務分支 |
| Model | `src/main/java/${BASE_PACKAGE_PATH}/model` | Java `record` / DTO（與 JdbcClient 自動 mapping） | API policy |
| Configuration | `src/main/java/${BASE_PACKAGE_PATH}/config`（如需） | `@Configuration`、Bean 宣告、共用基建 adapter | 功能專屬規則 |
| Entry | `src/main/java/${BASE_PACKAGE_PATH}/Application.java` | `@SpringBootApplication` 入口、`SpringApplication.run(...)` | 健康檢查以外的功能 endpoint 膨脹 |

### §1.1 App Entry Contract

- `Application.java` **必須** 標註 `@SpringBootApplication` 並暴露 `public static void main(String[] args)`。
- **`GET /health`**：**僅**基建探活；回應 **必須** 為 `{"status": "ok"}`，**禁止**累積產品行為。
- 本骨架 Controller 由 **`@RestController`** 直接回傳 Java `Map`／`record`，不額外封裝 envelope。

## §2 Dependency Direction

**允許**

```text
controller -> model
controller -> service
service    -> repository
service    -> model
repository -> model
config     -> infrastructure libraries
```

**禁止**

```text
model      -> controller
model      -> service
repository -> controller
service    -> controller
config     -> feature-specific services
```

---

## §3 Configuration And Dependency Injection

- 設定來自 **`application.yaml` / `application-test.yml` 與環境變數**；feature 層 **禁止** 硬編 URL／secret。
- `DataSource`／`JdbcClient` **必須** 經 Spring 容器注入（建構子注入優先）；**禁止** 在功能模組建立 `DataSource` singleton。
- 測試環境的 DB 連線 **必須** 經 `TestcontainersConfiguration`（`@ServiceConnection`）置換，**禁止** 連線本機資料庫跑 acceptance。

---

## §4 Persistence And Migration

- Schema SSOT：**`src/main/resources/db/migration/V{N}__xxx.sql`**（Flyway）。
- 啟動時 Flyway autoconfigure 會自動套用 `db/migration/` 下的 SQL；feature **禁止** 假設特定版本順序硬編在業務碼。
- 每一次持久化結構變更 **必須** 伴隨新的 `V{N+1}__...sql`；不得修改既有已套用之 migration（以新 revision 修正）。
- ORM 層採 **Spring `JdbcClient`**（非 JPA / Hibernate）；Java `record` 對應查詢結果，`JdbcClient.query(MyRecord.class)` 自動 mapping。
- 若新增 JPA 需求，須先在本檔修訂 §1 Layer Ownership 與 §4 條目。
