# Acceptance Runner Runtime

## acceptance runner command

```bash
./mvnw test
```

> 由 `RunCucumberTest`（`@Suite + @IncludeEngines("cucumber")`）作為 JUnit Platform 入口；`./mvnw test` 會跑 `RunCucumberTest`（Cucumber 整套 features，含 starter smoke `HealthCheck.feature`）。

## dry-run / discovery command

```bash
./mvnw test -Dcucumber.execution.dry-run=true
```

> Cucumber dry-run 不實際執行 step；只檢查所有 `.feature` 是否都有對應 step definition，可快速攔截 `UndefinedStepException` / `AmbiguousStepDefinitionException`。

## runner config file path

無單獨 `cucumber.properties` / `cucumber.options`；runner 設定集中在：

- `src/test/java/${BASE_PACKAGE_PATH}/RunCucumberTest.java`（`@Suite` 註解；`@SelectClasspathResource("features")` 指定 feature root）
- `src/test/java/${BASE_PACKAGE_PATH}/config/CucumberSpringConfiguration.java`（`@CucumberContextConfiguration` + `@SpringBootTest` + `@AutoConfigureMockMvc` + `@Import(TestcontainersConfiguration.class)`）

> Glue path 由 `cucumber-spring` 自動掃描 `RunCucumberTest` 所在 package；`steps/`、`config/`、`cucumber/` 子套件皆會被掃到。

## runtime feature glob

`src/test/resources/features/**/*.feature`

## step definition glob

`src/test/java/${BASE_PACKAGE_PATH}/steps/**/*.java`

## shared step glob（若有）

`src/test/java/${BASE_PACKAGE_PATH}/steps/common_then/**/*.java`

## report output path（若有）

無。預設 starter 不輸出 JUnit XML、HTML report；若專案啟用 Cucumber HTML / JSON / JUnit plugin，請在 `RunCucumberTest` 加 `@ConfigurationParameter(key = PLUGIN_PROPERTY_NAME, value = "html:target/cucumber-reports.html, json:target/cucumber.json")`，並在本檔明確補上實際路徑。

## runner visibility check 方法

1. 確認 `RunCucumberTest` 的 `@SelectClasspathResource("features")` 與 feature 實體位置一致。
2. 執行 `./mvnw test -Dcucumber.execution.dry-run=true`。
3. 若 dry-run 無法載入 feature 或 step definitions，先修 runner wiring（glue path、Spring DI、MockMvc 注入），不進入紅燈。

## stack-specific caveat / known limitations

- `cucumber.execution.dry-run=true` **不**會啟動完整 `@SpringBootTest` context（不會跑 Flyway / Testcontainers）；只能檢查 step matcher。
- Testcontainers 需要 Docker 可用；真正 acceptance runner command 可能因 Docker 未啟動而失敗，請先 `docker info` 或 `docker compose up -d`。
- 中文 feature 檔名依賴本機檔案系統與 JVM file encoding；`src/test/resources/` 預設以 UTF-8 載入。
- Spring Boot 4 拆分 starter 後，`MockMvc` 由 `spring-boot-starter-webmvc-test` + `@AutoConfigureMockMvc` 提供，**非**舊版 `spring-boot-starter-test`。
