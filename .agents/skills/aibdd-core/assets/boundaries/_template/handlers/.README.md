# 這個目錄放什麼？

每個 handler 的 runtime（實作期）履約文件，一個 handler 一檔，檔名等於 boundary 支援的 handler ID。內容寫該 handler 跑測試時讀寫哪些 context 欄位、何時該選用、禁止做什麼。plan-time 構造規則在 `scripts/part_to_dsl.py`。

# 哪裡會用到此目錄下的檔案？

在 .claude/skills/aibdd-red-execute：把 step 依 DSL entry 的 `handler` 分到某 handler 後，讀這裡對應的 `<handler-id>.md`（複製自 `handler.md` 並重新命名，如 `state-builder.md`）然後交叉比對 `aibdd-core/assets/boundaries/<your-boundary>/variants` 之中的 tech stack，來決定如何撰寫測試程式碼才會遵守 "某種handler x 某種技術variants" 的測試撰寫範式。
