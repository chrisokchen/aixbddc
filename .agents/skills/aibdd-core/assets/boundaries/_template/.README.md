# 這個目錄放什麼？

- 一個 boundary preset 的完整檔案家族範本：profile.yml（身分與規格綁定）、shared-dsl-template.yml（kickoff 種子）、scripts/（part-to-DSL plugin），以及選用的 handlers/、variants/、dsl-arrangement-rules/、sbe-form-lock/。

- 新增 boundary 時把整包複製成 boundaries/<your-boundary>/，將範本檔（如 variant.md、handler.md、rule.md、rule-type.tmpl）重新命名為實際名稱（如 python-e2e.md、state-builder.md 等），最後逐檔把文件內容中的 `<尖括號 placeholder>` 換成真值。每個子目錄各有 .README.md 說明自己的用途與下游消費者。

# 子規範

- profile.yml 宣告「這個端點的規格是什麼形狀」（state / operation / component specifier 各綁哪個 skill 與 format）。因此「規格用 DBML」「規格有持久化 Data」這類事都是某個 boundary 的配置事實，不是 AIBDD 的普世真相——換一個端點，這組 binding 就換一組值。

- 擴充只能用「新增或修改一個 boundary package」達成；boundary folder 以外的 skill 原始碼一律技術端點不可知，不因新增端點而被改動。新增端點時若發現非得改 folder 外的 skill 才能支援，那是那支 skill 不夠端點中立的訊號——要修的是那支 skill，而不是在它裡面加端點分支。
