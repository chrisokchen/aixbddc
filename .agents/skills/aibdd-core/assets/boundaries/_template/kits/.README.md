# 這個目錄放什麼？

一些 boundary-specific 的任務指示，通常是不同 boundary 之間在此任務下的指示大同小異，那為求方便，就先萃取出來，逐條放在 `kits/` 之中維護。


# 哪裡會用到此目錄下的檔案？

1. `how-to-look-up-dsl.md`
 1. 在 `/aibdd-red-execute`：`steps/implement-all-dsl-steps-in-feature-file.md` 步驟 2 禁止 agent 直接 `Read` / `Grep` 任何 `*.dsl.yml`；須改遵照 active boundary 的此檔指示，以 `dsl_cli query` 查詢該 Gherkin DSL Step 的完整 definition entries。
