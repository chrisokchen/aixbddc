# `/aibdd-tasks` Skill RCA

## Defect

`/aibdd-tasks` can generate `tasks.md`, but the generation boundary between deterministic structure and semantic fill is too blurry.

## Earliest Faulty Artifact

The earliest faulty artifact is the rendering architecture itself:

- `assets/templates/tasks-md.template.md` only defines the outer document shell
- there is no deterministic feature-phase scaffold artifact
- `SKILL.md` mixes feature-phase structure, GREEN allocation, and final markdown composition in one semantic-heavy flow

## Defect Types

- Deterministic / semantic boundary leak
- Template under-specification
- Missing intermediate scaffold gate

## Root Cause

The skill has a final-result validator, but no deterministic artifact that freezes:

- impacted feature order
- phase numbering
- fixed `RED / GREEN / Refactor` section skeleton

As a result, the model is forced to re-decide both structure and content in the same rendering step.

## Correction Direction

1. Keep one document-level template.
2. Add one feature-phase template.
3. Add a Python scaffold script that deterministically produces ordered feature phase metadata.
4. Shrink semantic fill to infra / integration plus GREEN waves.
5. Add deterministic validation for scaffold shape in addition to final `tasks.md`.

## Risk

Low.
The refactor mostly clarifies responsibility boundaries and increases deterministic coverage without changing the user-facing contract of `tasks.md`.
