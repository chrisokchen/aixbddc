# Benchmark 03 — 審稿後自動上架

## 埋藏陷阱

- 兩位審稿人均通過後，系統**自動上架**並**自動選擇**展示策略。
- 庫存充足／偏低／為零三種策略**可獨立命名**，易各拆一 feature。
- 「進入已發布」「執行發布」易成純階段轉換 feature。

## 需求原文

內容營運平台：編輯可**建立內容草稿**並**提交審稿**，系統指派兩位審稿人。

每位審稿人可**完成審閱**（通過或要求修改）。若要求修改，編輯修改後再次提交審稿，審稿人重新審閱。

當**兩位審稿人均標記通過**後，系統**自動上架**該內容，並依即時庫存**自動選擇**展示策略（三種：庫存充足／偏低／為零）。編輯與審稿人皆無「上架」「選擇策略」按鈕。

已上架內容可由具權限者**下架內容**（使用者主動操作）。

## 預期產出之 feature file

- `建立內容草稿.feature`
- `提交審稿.feature`
- `完成審閱.feature`
- `下架內容.feature`

## 不應出現（常見誤拆）

- `系統自動上架.feature`、`執行發布.feature`、`進入已發布.feature`
- `展示策略庫存充足.feature`、`展示策略庫存偏低.feature`、`展示策略庫存為零.feature`、`計算展示策略.feature`

上架與策略選擇為審稿全通過後之編排副作用；三種策略為 **Scenario**，不另開 feature。
