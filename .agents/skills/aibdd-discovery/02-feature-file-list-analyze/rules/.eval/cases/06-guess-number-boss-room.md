# Benchmark 06 — 1A2B 猜數字魔王房

## 埋藏陷阱

- **玩家都輪完後**輪到魔王，魔王四種行動（群傷／單體／自療／改密碼）**可獨立命名**，易拆成 `魔王執行回合.feature` 或四個技能 feature。
- 開房與加入同房號、產生魔王、進入階段一／二、全員設完密碼後開局，易拆成多個狀態或系統 feature。
- 1A2B 判定、加權傷害、爆擊、跳過出局玩家、機率抽技能屬**內建系統處理**，易與玩家猜測並列。

## 需求原文

一個 1A2B 猜數字魔王遊戲。

遊戲大廳主畫面簡潔：輸入四位數字房號，按「開房」。若該房號不存在則開新房；若已存在則加入該房。房間最多 10 人；開房者為房主（P1），加入者依序為 P2～P9。

房客須先按「準備」；房主在房客皆已準備且至少 2 人時可按「開始遊戲」。遊戲開始後系統**自動**產生魔王並進入 1A2B 對戰；魔王血量與血量上限為玩家人數 × 1000。玩家一隊、魔王一隊。

**階段一（設密碼）**：密碼為四位不重複數字（0–9）；魔王密碼由系統合法隨機產生，玩家平行各自設好密碼。當**全部玩家都設好密碼**後，系統**自動**進入階段二。

**階段二（對戰）**：兩隊輪流，先玩家隊。玩家隊內 P1～P9 輪流，出局者跳過但仍可觀戰。每位玩家回合僅做一件事：輸入四位不重複數字猜魔王密碼；依 1A2B 規則（A＝位置與數字皆對，B＝數字對位置錯）與加權公式（含爆擊機率）對魔王造成傷害。若魔王血量歸零則玩家勝利。

**玩家隊全員本輪皆已出手後**，系統**自動**進入魔王回合（無「開始魔王回合」按鈕）。魔王依機率選擇行動：40% 群傷攻擊、30% 單體攻擊、20% 自我治療、10% 改變密碼；若任一位玩家曾對魔王猜出 4A，則魔王回合**必定**改變密碼。四種行動效果：群傷—對 3 位玩家各猜 1 次並造成傷害；單體—對 1 位玩家猜 3 次各造成傷害；自療—對 3 位玩家各猜 1 次，將傷害數值轉為自療且不超過上限；改密碼—直接更換魔王密碼。魔王造成傷害後若玩家死亡則出局；**全部玩家出局**則失敗。

## 預期產出之 feature file

- `開房或加入遊戲房.feature`
- `玩家準備.feature`
- `房主開始遊戲.feature`
- `玩家設置密碼.feature`
- `玩家猜測魔王數字.feature`

## 不應出現（常見誤拆）

- `開設遊戲房.feature`、`加入遊戲房.feature`（同房號單一使用者意圖，應合併）
- `產生魔王.feature`、`進入階段一.feature`、`進入階段二.feature`
- `魔王執行回合.feature`、`群傷攻擊.feature`、`單體攻擊.feature`、`自我治療.feature`、`改變密碼.feature`
- `計算1A2B.feature`、`計算傷害.feature`、`判定玩家勝利.feature`、`判定遊戲失敗.feature`

產生魔王、進入階段二併入 `房主開始遊戲`、`玩家設置密碼` 之 **postcondition**；玩家全輪完後之魔王回合與四種技能為 `玩家猜測魔王數字` 編排副作用與 **Scenario**，不另開 feature。
